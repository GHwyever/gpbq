## 四、视频生成规则

### 4.1 API 配置

| 项目 | 值 |
|------|-----|
| Endpoint | `https://clmm-mall.top/v1/video/generations` |
| 状态查询 | `https://clmm-mall.top/v1/video/generations/{task_id}` |
| API Key | `sk-3lvVLWHpqH5NL6uF68j5WCsj6bml1O6iQJV0uJKbudqhCSSp` |

### 4.2 模型选择指南

| 模型 | 适用场景 | 预扣 | 面部质量 |
|------|----------|:--:|:--:|
| `mg-seedance2.0 -720p fast-15s` | 场景/动作/远景 | ¥20 | 差，面部推镜必扭曲 |
| `gu-seedance2.0-pro 720p-15s` | **面部特写/推镜** | ¥3.78 | 好 |
| `mg-seedance2.0 -720p pro-15s` | 高画质面部特写 | ¥6.20 | 最好 |
| `sh-seedance2.0-pro 720p-nv-15s` | 备选pro | ¥5.80 | 好 |

**铁律**：面部特写、推镜镜头**必须用 pro 模型**，fast 模型仅用于远景/场景/动作。

### 4.3 预扣费机制

- 预扣≠实际扣费，生成完后按实际用量扣，余款退回
- 提交前必须确认余额 ≥ 预扣门槛
- 定价：约 ¥0.27/秒

### 4.4 参考图原则

**每条视频必须传入**：
- `first_frame`：首帧 CDN URL（必传）
- `reference_images`：1-4 张参考图 CDN URL（必传，至少包含角色四视图）
- 场景参考图建议同时传入

### 4.5 审核规避

**已知触发词**（clmm-mall.top 社区规范）：

| 触发词 | 替代 |
|--------|------|
| 濒临死亡 | 气息微弱 / 沉睡 |
| 瞳孔涣散 | 眼神迷离深邃 |
| 奄奄一息 | 极度虚弱 |
| 血 / 鲜血 | 暗色液体 / 暗红 |
| 赤裸上身 | 穿白色单薄内衫，衣襟微敞 |
| 七窍流血 | 暗色液体从七窍涌出 |
| 杀 / 死 | 回避，用意境表达 |

**所有提示词中涉及"濒死""死亡"等词一律替换。**

## 八、生产流程（逐镜执行）

```
第1步：核对分镜 → 确认该镜模式（首帧+参考图 / 首帧+尾帧）
第2步：生成首帧 → POST /v1/media/generate（GPT Image 2）
       ├── 传入角色四视图 + 基准脸 + 场景参考
       └── 下载 → 04-竖屏首尾帧/EP0X/
第3步：生成尾帧 → 同上（仅首尾帧模式需要）
第4步：更新分镜文档状态（待生成→✅）
第5步：推送首尾帧到 GitHub CDN → 00-原文资料/cdn/
第6步：生成视频 → POST /v1/video/generations（Seedance 2.0）
       ├── 面部特写/推镜 → pro 模型
       ├── 远景/场景/动作 → fast 模型
       └── 必传 first_frame + reference_images
第7步：下载视频 → 06-竖屏视频/EP0X/
第8步：更新分镜文档状态（视频生成→✅）
第9步：审核视频质量 → 面部一致性/运镜效果/画面扭曲
       ├── 通过 → 下一镜
       └── 不通过 → 调整参数重试
```

## 九、关键教训

| 编号 | 教训 | 影响 |
|:--:|------|------|
| 1 | fast 模型面部推镜必扭曲，必须用 pro | 浪费 ¥8 + 3次尝试 |
| 2 | 参考图必须传满，否则角色/场景不一致 | 视频不可用 |
| 3 | "濒临死亡""瞳孔涣散"触发审核 | 2次失败，扣余额 |
| 4 | 提示词中必须有"古装"标识，否则生成现代装 | 角色风格错误 |
| 5 | 苏然面部锚点必须包含"年轻中国男子苏然"全名 | 角色一致性 |
| 6 | GitHub raw URL 需要仓库 Public | 半天排查 |
| 7 | 预扣费≠实际扣费，余额需≥预扣门槛 | 多次提交失败 |
| 8 | 生图参考图必须覆盖角色+场景，缺一不可 | 冰窖主空间图角色不一致 |
| 9 | 视频提示词需规避审核敏感词，但保留意境 | 分镜文案需两套（内部/提交） |
